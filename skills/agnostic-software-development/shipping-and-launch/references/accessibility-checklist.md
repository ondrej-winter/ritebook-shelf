# Pre-Launch Accessibility Checklist

Use this checklist before shipping browser-facing UI changes.

## Required checks

- Confirm primary flows work with keyboard navigation.
- Confirm focus order and visible focus states are clear.
- Confirm interactive controls have accessible names.
- Confirm form labels, errors, and descriptions are associated with inputs.
- Confirm text and meaningful icons meet contrast expectations.
- Confirm state is not communicated by color alone.
- Confirm automated accessibility checks pass when available.

## Release evidence

- Record browser, viewport, and assistive checks performed.
- Document any known accessibility limitation and owner.
- Confirm accessibility issues are not hidden by launch pressure.
